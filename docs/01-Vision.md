## 1.1 Название проекта

Название проекта: **FleetOps**

## 1.2 Описание проекта

FleetOps — информационная система управления автопарком, предназначенная для автоматизации процессов
эксплуатации транспортных средств и управления связанными бизнес-процессами организации.

Проект представляет собой серверное приложение, реализованное в виде REST API.

Система предназначена для централизованного хранения информации об автопарке, управления транспортными средствами,
сотрудниками, техническим обслуживанием, ремонтами, маршрутами и другими объектами предметной области.

Архитектура проекта предусматривает возможность дальнейшего расширения функциональности
без существенного изменения существующей структуры системы.

## 1.3 Цель проекта

Основной целью проекта является разработка современной информационной системы,
обеспечивающей автоматизацию процессов управления автопарком.

Система должна сократить количество ручных операций, повысить качество хранения информации,
обеспечить контроль основных бизнес-процессов и предоставить удобные средства управления транспортными средствами.

## 1.4 Для кого предназначен проект

Система предназначена для организаций, использующих собственный автопарк независимо от сферы деятельности.

Предполагаемыми пользователями системы являются:

- транспортные компании;
- логистические организации;
- службы доставки;
- службы такси;
- строительные предприятия;
- производственные организации;
- государственные учреждения;
- предприятия с корпоративным транспортом.

## 1.5 Какие проблемы решает проект

Разрабатываемая система направлена на решение следующих задач:

- централизация хранения информации;
- автоматизация управления автопарком;
- сокращение количества ручных операций;
- снижение вероятности возникновения ошибок;
- контроль эксплуатации транспортных средств;
- контроль технического обслуживания и ремонтов;
- учет расхода топлива и пробега;
- получение актуальной отчетности;
- повышение эффективности использования автопарка.

## 1.6 Основные возможности

В первой версии проекта реализованы следующие функции:

- управление пользователями;
- управление ролями;
- управление транспортными средствами;
- управление водителями;
- управление подразделениями;
- учет технического обслуживания;
- учет ремонтов;
- учет пробега;
- учет расхода топлива;
- управление маршрутами;
- управление рейсами;
- хранение документов;
- формирование отчетов;
- ведение истории изменений.

## 1.7 Цели разработки

При разработке проекта ставятся следующие цели:

- применение современных технологий разработки программного обеспечения;
- проектирование масштабируемой архитектуры;
- создание качественной структуры базы данных;
- обеспечение возможности дальнейшего расширения функциональности;
- получение практического опыта разработки корпоративных информационных систем;
- создание полноценного портфельного проекта.

## 1.8 Используемые технологии

В проекте применяются следующие технологии:

- C#;
- Blazor;
- ASP.NET Core Web API;
- Entity Framework Core;
- SQL Server;
- JWT Authentication;
- Swagger (OpenAPI);
- AutoMapper;
- FluentValidation;
- Docker;
- Git;
- GitHub.

## 1.9 Границы проекта

В рамках первой версии проекта реализованы, непосредственно связанные с управлением автопарком.

В первую версию входят:

- учет транспортных средств;
- учет сотрудников;
- управление пользователями;
- управление ролями;
- учет технического обслуживания;
- учет ремонтов;
- учет пробега;
- учет расхода топлива;
- управление маршрутами;
- управление рейсами;
- формирование отчетности.

В первую версию проекта не входят:

- GPS/ГЛОНАСС-мониторинг;
- мобильное приложение;
- интеграция с бухгалтерскими системами;
- интеграция с государственными информационными системами;
- управление складом запасных частей;
- телеметрия транспортных средств;
- автоматическое построение маршрутов;
- искусственный интеллект для анализа данных.

Перечисленные возможности могут быть реализованы в последующих версиях системы.
